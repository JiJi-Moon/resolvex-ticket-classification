System Architecture

ResolveX follows a layered architecture combining AI-driven intelligence with structured workflow governance to automate enterprise-grade ticket management.

🏗 High-Level Architecture

1. Presentation Layer

Web-based interface built using Flask and Tailwind CSS. Enables user interaction, ticket submission, and structured visualization of workflow status.

2. AI Processing Layer

NLP models analyze user input, perform classification, extract entities, and determine intelligent priority levels.

3. Ticket Generation Engine

Converts AI outputs into structured JSON objects mapping title, description, department, and priority fields.

4. Workflow & Governance Layer

Enforces lifecycle transitions (Open → In Progress → Resolved → Closed) with strict role-based validation.

🤖 AI Processing Pipeline

Step 1: Input Preprocessing

Cleans and normalizes text by removing greetings, noise, formatting issues, and irrelevant content.

Step 2: Text Classification

NLP classification models assign tickets to predefined departments using a fine-tuned BERT model.

Step 3: Named Entity Recognition

Extracts key entities including services, systems, and error codes.

Step 4: Hybrid Priority Assignment

Combines rule-based keyword detection and tuned Linear SVM model to determine urgency level.
